## Features Shipped

### 1. v1.1 — Real Options Chains
**What:** Live SPX options data with Greeks flowing into every agent's analysis and trade card.

- Tradier API client (async, sandbox + production)
- OptionContract and OptionsSnapshot data models
- Delta-based strike selection for agent prompts
- Greeks (delta, gamma, theta, vega, IV) in trade cards and Telegram alerts
- Graceful fallback when no API key configured

### 2. v1.2 — Outcome Tracking
**What:** Every signal is tracked. After 2 hours or EOD, the system resolves WIN/LOSS/SCRATCH with P&L and feeds results back to memory.

- OutcomeTracker service with async signal resolution
- P&L computation (bull and bear directions)
- AOMS memory feedback loop — the swarm learns from its history
- Signal History panel in web dashboard
- GET /api/signals and /api/stats REST endpoints
- Outcome notifications via Telegram and Slack

### 3. v3.0-lite — Custom Agents
**What:** Define your own agent personas in YAML. Up to 6 custom agents merge into the swarm alongside the base 24.

- config/custom_agents.yaml with documented template
- Forge loads, validates, and caps at 30 total agents
- Unknown tribes fall back to fast_local model
- REST API: POST/DELETE /api/agents/custom
- Full validation: duplicate IDs, required fields, cap enforcement

### 4. UI/UX Overhaul (10 Improvements)
**What:** 10 UI/UX agents debated the dashboard and produced a prioritized list. All 10 shipped, code-reviewed by 4 parallel agents.

1. **Stats bar** — Win Rate, Avg P&L, W/L record, PCR, ATM IV
2. **WCAG contrast fix** — text-dim and text-muted bumped for accessibility
3. **Options data in stats bar** — PCR + ATM IV live from market context
4. **Greeks in trade card** — Strike, Delta, Premium, IV display
5. **Toast notifications** — WIN/LOSS/cycle complete/error slide-in toasts
6. **Cycle timer** — Elapsed during cycle, countdown to next
7. **Skeleton loading** — Pulse animations during cycle
8. **Canvas performance** — Idle 10fps throttle, cached dot grid, no shadowBlur
9. **Mobile tab navigation** — Bottom tab bar (Network/Signal/Debate/History)
10. **Keyboard shortcuts** — C=cycle, 1/2/3=panels, ?=help overlay

Post-review fixes: CycleTimer interval leak, DPR dot grid blit, aria-labels, dialog semantics, mobile layout overlap.

### 5. Asymmetric Gamma Scalping Strategy Engine
**What:** The system now trades like you do. Premium-targeted strike selection, time-of-day awareness, multi-leg structures.

- **Morning:** $5-$8 OTM straight calls/puts, target 3x+ (STRAIGHT)
- **Afternoon:** $0.50-$2 deep OTM lottos, target 5-10x (LOTTO)
- **High VIX (>20):** Vertical debit spreads, capped risk (VERTICAL)
- **Choppy/range:** Iron condors, sell premium (IRON_CONDOR)
- Strategy selector runs before the LLM — picks the structure from real chain data
- Trade card shows strategy badge, multi-leg structures, R:R ratios

### 6. Schwab Real-Time Data + Morning Briefing + Scheduler
**What:** Replaced yfinance with Schwab Trader API. 1,138 live SPX option contracts. Automated daily schedule with everything pushed to Telegram.

**Data stack:**
- Schwab: real-time L1 quotes ($SPX, $VIX, /ES futures), full options chain
- Tradier: options chain fallback
- yfinance: quote fallback only

**Morning briefing (8:00 AM ET):**
- ES futures overnight move, VIX, previous SPX close
- Regime forecast + strategy playbook (AM/PM/avoid/sizing)
- Key levels from options chain (support, resistance, gamma pin)

**Automated scheduler:**
- 8:00 AM: morning briefing → Telegram
- 9:35 AM: opening swarm cycle → trade card → Telegram
- 11:30 AM: midday cycle
- 2:00 PM: afternoon cycle (lotto mode auto-activates)
- 3:45 PM: close + daily summary (W/L/P&L)

**CLI:**
- `python -m swarmspx.cli schedule` — start scheduler daemon
- `python -m swarmspx.cli briefing` — one-off briefing
- `python -m swarmspx.cli run` — single cycle
- `python -m swarmspx.cli web` — dashboard
